# 리스크 및 거버넌스 운영 가이드 (Risk & Governance Guidelines)

자동 투자 조직이 예기치 못한 시장 상황이나 내부 시스템 오류로부터 생존하기 위한 **3중 안전장치**를 공식 편입하여 운영합니다.

## 1. Global Risk Agent (전사적 리스크 감시)
- **역할**: Investment Agent들의 개별 리스크(특정 종목 몰빵 등)가 합쳐져 펌 전체의 치명적 리스크로 번지는 것을 방지.
- **감시 항목 (연속 모니터링)**:
  - 펌 전체의 Portfolio Value at Risk (VaR)
  - 펌 전체의 Maximum Drawdown (MDD)
  - 특정 자산군/섹터에 대한 과도한 상관관계 노출 (Correlation Risk)
- **액션**:
  - 리스크 한도(Hard Limit)의 80% 도달 시 Manager Agent에게 **경고(Warning)** 발송, 신규 진입 제한.
  - 리스크 한도 100% 도달 시 **Kill Switch** 가동.

## 2. Kill Switch (비상 정지 시스템)
- **역할**: 특정 조건 충족 시 즉각적으로 모든 매매를 중단하고 포지션을 청산(또는 동결)하는 강제 권한 시스템.
- **가동 조건 (Trigger Conditions)**:
  1. **시장 붕괴 (Market Crash)**: VIX 지수(변동성)가 사전에 정의된 극단적 수치(예: 40 이상)를 돌파하거나, 주요 지수가 하루 X% 이상 급락할 때.
  2. **API/인프라 장애**: 증권사 브로커, 암호화폐 거래소 API, 시세 데이터 API 등에서 연속 N회 이상 데이터 수신 실패 시. (환각 상태 매매 방지)
  3. **누적 손실 하드 리미트**: 시스템 전체 자본금 기준 일간/주간/월간 손실이 허용 임계치를 초과할 때.
  4. **Rogue Agent (로직 이탈)**: 특정 Investment Agent의 행동이 그들의 `strategy.md`와 수학적으로/논리적으로 완전히 불일치할 때.
- **실행 절차**:
  1. 모든 Agent 프로세스 즉각 Suspend(정지).
  2. 보유 중인 모든 포지션에 대한 시장가 청산 로직 가동 (사용자 설정에 따라 동결(Hold) 옵션 적용 가능).
  3. Human Owner에게 슬랙/텔레그램 등 긴급 알림 발송.

## 3. Shadow Agent (사후 감사 및 Audit)
- **역할**: 의사결정의 투명성과 개선점을 도출하기 위한 **과거 회고용 로거(Logger)**이자 분석가.
- **운영 방식**:
  - 실시간 매매에는 관여하지 비동기적으로(Asynchronous) 작동.
  - 매일 자정, Manager Agent의 해고/채용 기록(`hr_records.md`)과 Investment Agent들의 매매 기록(`trades.md`)을 읽음.
  - **분석 항목**: "이 매매는 당시 시장 상황과 `strategy.md`에 비추어 볼 때 합리적이었는가?"
  - **산출물**: 감사 리포트를 생성하여 Human Owner가 에이전트들의 사고방식을 검증하고 프롬프트를 디버깅하는 데 활용.
